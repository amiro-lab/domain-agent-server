# 하네스(Harness) vs 에이전트(Agent)

## 1. 기본 정의

**하네스(harness)** = LLM을 둘러싼 **실행 환경/런타임**
- 결정론적 코드. LLM이 아님.
- 도구 호출 실행, 권한 프롬프트, 훅 발동, 파일 상태 추적, 컨텍스트 압축, 세션 관리, system-reminder 주입 등
- "루프를 돌리는 프로그램"

**에이전트(agent)** = **LLM 본체 + 프롬프트 + 도구셋 + 컨텍스트**
- 비결정론적. 판단/생성하는 두뇌.
- 시스템 프롬프트, 사용 가능한 tool 목록, 자기 컨텍스트 윈도우를 가짐.
- 메인 에이전트(Claude Code 본인)와 서브에이전트(`Agent` 툴로 스폰되는 Explore/Plan/general-purpose 등)로 나뉨.

## 2. 정확한 층위

```
┌─ 하네스 (Claude Code CLI 본체, 1개 프로세스) ────┐
│  공유 서비스: 권한 / 훅 / 파일 상태 / Bash 실행   │
│                                                  │
│   ┌─ 메인 에이전트 ─┐                            │
│   │ Opus 4.7       │                            │
│   │ 풀 툴셋        │                            │
│   └────┬───────────┘                            │
│        │ Agent 툴 호출                           │
│        ▼                                         │
│   ┌─ 서브에이전트 (Explore/Plan/...) ─┐          │
│   │ 같은 모델 or 다른 모델            │          │
│   │ 제한된 툴셋                       │          │
│   │ 자기 컨텍스트 윈도우              │          │
│   └───────────────────────────────────┘          │
└──────────────────────────────────────────────────┘
```

- **하네스** = 1개 (프로그램 자체, 싱글턴)
- **에이전트** = 하네스가 인스턴스화한 *역할/구성*
- 하네스는 **여러 에이전트를 동시에 호스팅**할 수 있음

> "에이전트 = 하네스 + 모델"은 느슨한 정의로는 맞지만, 엄밀히는
> **에이전트 = 모델 + 프롬프트 + 툴셋 + 컨텍스트 (= 하네스 안의 한 슬롯)**.

## 3. 서브에이전트는 자기 하네스를 가지는가?

**아니, 하네스는 1개를 공유함.** 서브에이전트는 같은 하네스 안에 새로운 *에이전트 슬롯*만 추가로 띄우는 거.

### 같이 쓰는 것 (하네스 레벨)
- 권한 시스템 (서브에이전트 툴 호출도 같은 권한 룰 적용)
- 훅 (settings.json hooks)
- Bash 작업 디렉토리, 환경변수
- 파일시스템 접근

### 따로 가지는 것 (에이전트 레벨)
- **컨텍스트 윈도우** → "메인 컨텍스트 보호"용으로 서브에이전트 쓰는 이유
- **시스템 프롬프트** (Explore는 read-only 강조, Plan은 설계 강조 등)
- **툴셋** (Explore는 Edit/Write 없음)
- **모델** (서브마다 다른 모델 지정 가능)
- **대화 히스토리** — 서브에이전트는 메인 대화를 못 봄. prompt를 잘 써줘야 함.

## 4. 실용적 함의

| 상황 | 어디서 처리되는가 |
|------|-----------------|
| "X할 때마다 자동으로 Y" 같은 자동화 | **하네스** (settings.json hooks). 에이전트한테 약속받아도 다음 세션엔 잊어버림. |
| 추론·코드 작성·판단 | **에이전트** |
| 서브에이전트가 파일 수정 | 메인의 "Read 후 Edit" 추적 상태와 동기화됨 (같은 하네스) |
| 훅 발동 | 서브에이전트 툴 호출에도 발동됨 |
| 서브에이전트가 알아낸 사실 | 메인이 자동으로 모름. **최종 리턴 메시지로만** 전달됨 |

## 5. 비유

- **하네스 = OS**
- **에이전트 = 프로세스**
- **서브에이전트 = 자식 프로세스**

같은 OS 위에서 돌지만 메모리 공간(컨텍스트)은 분리.
